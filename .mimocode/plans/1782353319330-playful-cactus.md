# 项目优化计划 - 我的回忆 (My Memory)

## 项目概述
Unity 恐怖游戏项目，基于 Five Nights at Freddy's 风格。当前处于早期开发阶段，框架已搭建但存在多处代码质量问题和未完成的功能。

---

## 一、严重问题（会导致构建失败）

### 1. FaceCheck.cs 引用了 UnityEditor 命名空间
- **文件**: `Assets/FaceCheck.cs:4`
- **问题**: `using static UnityEditor.Progress;` 会在非编辑器构建时导致编译错误
- **修复**: 删除该行无用引用

### 2. Build Settings 中没有场景
- **问题**: `EditorBuildSettings.m_Scenes: []` 为空，打包会失败
- **修复**: 将 `SampleScene.unity` 添加到 Build Settings

---

## 二、性能优化

### 3. PlayerController 每帧 GetComponent
- **文件**: `Assets/Script/Player/PlayerController.cs:42`
- **问题**: `playerCamera.GetComponent<PlayerMouse>().MouseMove()` 每帧调用 GetComponent
- **修复**: 在 Start 中缓存 PlayerMouse 引用

### 4. FaceCheck 重复 GetComponent
- **文件**: `Assets/FaceCheck.cs:25-27`
- **问题**: `OnTriggerEnter` 中调用了两次 `GetComponent<ItemInterface>()`
- **修复**: 只调用一次并缓存结果

### 5. UIManager.GetPanel 每帧调用
- **文件**: `Assets/Script/Player/PlayerController.cs:72,89`
- **问题**: 跑步和耐力恢复时每帧通过字符串查找面板
- **修复**: 在 Start 中缓存 GameSencePanel 引用

---

## 三、代码质量问题

### 6. 空方法和死代码
- `PlayerController.FixedUpdate()` - 空方法，可删除
- `StartScript.Update()` - 空方法，可删除
- `FaceCheck.Update()` - 空方法，可删除
- `MoveState.cs` - 完全空的 MonoBehaviour 存根
- `MonoSingleton.Init()` - 空方法

### 7. 命名拼写错误
- `GameSencePanel` → 应为 `GameScenePanel`（"Sence" 是拼写错误）
- 涉及文件: `GameSencePanel.cs`, `PlayerController.cs`, `StartScript.cs`

### 8. 未使用的 using 引用
- `PlayerController.cs:4`: `using static UnityEngine.GraphicsBuffer;` 未使用
- `PlayerController.cs:1-2`: `System.Collections` 和 `System.Collections.Generic` 未使用
- `EventCenter.cs:3`: `Unity.VisualScripting` 未使用
- 多个文件有冗余的 `System.Collections` 引用

### 9. 注释掉的代码
- `PlayerController.cs:99,113`: 注释掉的 Lerp 代码
- `CachePoolMgr.cs:25-29`: 注释掉的资源加载代码

---

## 四、架构问题

### 10. FSM 框架未接入
- `PlayerFsmState.Start()` 中 `fsm.AddState()` 被注释掉
- `IdleState` 实现了 `IState` 但 `OnUpdate` 为空
- `MoveState` 是 MonoBehaviour 而非实现 `IState`
- **建议**: 要么完成 FSM 接入，要么暂时移除避免混淆

### 11. EventCenter 未使用
- `E_EventType` 枚举为空，没有任何事件定义
- **建议**: 开始定义游戏事件（如 PlayerDead, DoorOpened 等）

### 12. CachePoolMgr 资源加载未实现
- `GetObj` 中资源加载被注释掉
- **建议**: 集成 Resources.Load 或 Addressables

---

## 五、单例模式混乱

### 13. 三种不同的单例实现
- `MonoSingleton<T>` - 使用 `Ins` 属性
- `BaseManager<T>` - 使用 `Instance` 属性
- `UIManager` - 手动实现的单例，使用 `Instance` 属性
- `PlayerFsmState` - 手动实现的单例
- **建议**: 统一使用 `MonoSingleton<T>` 或 `BaseManager<T>`

---

## 六、优化优先级

| 优先级 | 编号 | 问题 | 影响 |
|--------|------|------|------|
| P0 | 1 | FaceCheck UnityEditor 引用 | 构建失败 |
| P0 | 2 | Build Settings 无场景 | 打包失败 |
| P1 | 3,4,5 | 每帧 GetComponent/字符串查找 | 运行时性能 |
| P1 | 6 | 空方法/死代码 | 代码整洁 |
| P2 | 7 | 拼写错误 | 可维护性 |
| P2 | 8 | 未使用的 using | 代码整洁 |
| P2 | 9 | 注释掉的代码 | 代码整洁 |
| P3 | 10-13 | 架构问题 | 长期可维护性 |

---

## 建议执行顺序

1. **修复构建错误** (P0) - 删除 FaceCheck 的 UnityEditor 引用，添加场景到 Build Settings
2. **性能优化** (P1) - 缓存组件引用，删除空方法
3. **代码清理** (P2) - 修正拼写，清理 using，删除注释代码
4. **架构改进** (P3) - 根据开发计划决定是否完成 FSM/EventCenter 接入

---

## 验证方式

1. 在 Unity 中打开项目，确认无编译错误
2. 运行游戏，确认玩家控制、门交互、UI 显示正常
3. 使用 Profiler 确认无每帧 GC 分配
4. 尝试 Build 项目，确认能成功打包
