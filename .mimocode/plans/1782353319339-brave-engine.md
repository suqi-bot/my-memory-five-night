# 恐怖氛围系统实现计划

## 项目概述
游戏名称：《鬼屋试睡员》
类型：现代写实第一人称时间分段式恐怖生存游戏（FNAF-like）
引擎：Unity 2022.3 C#

## 目标
实现恐怖氛围系统，增强游戏的沉浸式恐怖体验。

---

## 系统架构

### 1. 恐怖氛围管理器 (HorrorAtmosphereManager)
**路径**: `Assets/Script/Horror/HorrorAtmosphereManager.cs`

核心职责：
- 统一管理所有恐怖氛围效果
- 根据游戏阶段（黄昏/午夜）调整氛围强度
- 协调音效、视觉、事件三大子系统

```csharp
public class HorrorAtmosphereManager : MonoSingleton<HorrorAtmosphereManager>
{
    // 氛围强度 0-1（黄昏=0.2, 午夜=1.0）
    public float AtmosphereIntensity { get; private set; }
    
    // 鬼怪接近度 0-1（用于心跳、音效变化）
    public float GhostProximity { get; set; }
    
    // 理智值（影响视觉效果）
    public float Sanity { get; set; }
}
```

---

### 2. 环境音系统 (AmbientAudioSystem)
**路径**: `Assets/Script/Horror/AmbientAudioSystem.cs`

功能：
- **持续低鸣**：低频嗡嗡声，作为背景氛围
- **风声/管道声**：随机间隔播放，制造紧张感
- **时钟滴答声**：仅在收集阶段，随时间加速
- **心跳声**：鬼怪接近时渐强，使用AudioManager的循环音效

实现方式：
```csharp
public class AmbientAudioSystem : MonoBehaviour
{
    // 音频ID常量（对应AudioDataConfig）
    private const string AMBIENT_HUM = "ambient_hum";
    private const string AMBIENT_WIND = "ambient_wind";
    private const string AMBIENT_CLOCK = "ambient_clock";
    private const string AMBIENT_HEARTBEAT = "ambient_heartbeat";
    
    // 心跳声随鬼怪接近度变化
    public void UpdateHeartbeat(float proximity);
}
```

---

### 3. 恐怖视觉效果系统 (HorrorVisualEffects)
**路径**: `Assets/Script/Horror/HorrorVisualEffects.cs`

功能：
- **镜头晃动**：随机轻微震动，鬼怪接近时加强
- **屏幕闪烁**：UI Image覆盖层实现全屏闪烁
- **Vignette效果**：使用UI Image实现暗角效果
- **屏幕噪点**：使用噪点材质实现颗粒感

实现方式（不依赖Post Processing包）：
```csharp
public class HorrorVisualEffects : MonoBehaviour
{
    // 镜头晃动
    private Vector3 originalCameraPosition;
    private float shakeIntensity;
    
    // 屏幕效果UI层
    private Canvas effectsCanvas;
    private Image vignetteImage;
    private Image flickerImage;
    private RawImage noiseImage;
    
    // 根据理智值调整效果
    public void UpdateEffects(float sanity, float ghostProximity);
    
    // 镜头晃动
    public void TriggerCameraShake(float intensity, float duration);
}
```

视觉效果实现方案：
1. **镜头晃动**：直接修改Camera的localPosition
2. **屏幕闪烁**：使用全屏UI Image，调整alpha实现闪烁
3. **Vignette效果**：使用四角渐变的UI Image
4. **屏幕噪点**：使用噪点纹理的RawImage，随机UV偏移
```

---

### 4. 恐怖事件系统 (HorrorEventSystem)
**路径**: `Assets/Script/Horror/HorrorEventSystem.cs`

功能：
- **灯光闪烁**：随机触发，频率随氛围强度增加
- **门自动关闭**：远处门随机关闭，伴随音效
- **远处脚步声**：3D空间音效，位置随机
- **镜子黑影**：镜面反射中出现黑影
- **物品掉落**：场景中物品随机掉落

实现方式：
```csharp
public class HorrorEventSystem : MonoBehaviour
{
    // 事件配置
    [SerializeField] private HorrorEventConfig eventConfig;
    
    // 事件触发器列表
    private List<IHorrorEvent> activeEvents;
    
    // 根据游戏阶段触发不同事件
    public void TriggerRandomEvent(GamePhase phase);
}
```

### 5. 恐怖事件配置 (HorrorEventConfig)
**路径**: `Assets/Script/Horror/HorrorEventConfig.cs` (ScriptableObject)

```csharp
[CreateAssetMenu(fileName = "HorrorEventConfig", menuName = "Horror/Event Config")]
public class HorrorEventConfig : ScriptableObject
{
    public List<HorrorEventData> events;
}

[Serializable]
public class HorrorEventData
{
    public string eventId;
    public HorrorEventType type;
    public float minIntensity;  // 触发所需的最低氛围强度
    public float maxIntensity;  // 触发所需的最高氛围强度
    public float cooldown;      // 冷却时间
    public float probability;   // 触发概率
}
```

## 实现步骤

### Phase 1: 核心框架 (1-2天)
1. 创建 `HorrorAtmosphereManager` 主管理器
2. 创建 `AmbientAudioSystem` 环境音系统
3. 在 AudioDataConfig 中添加恐怖氛围音效配置

### Phase 2: 视觉效果 (1-2天)
1. 创建 `HorrorVisualEffects` 视觉效果系统
2. 使用Unity内置方式实现（不依赖Post Processing包）
3. 实现镜头晃动、屏幕闪烁、Vignette效果（使用OnRenderImage或Camera脚本）

### Phase 3: 事件系统 (2-3天)
1. 创建 `HorrorEventSystem` 事件系统
2. 创建 `HorrorEventConfig` 配置文件
3. 实现灯光闪烁、门自动关闭、远处脚步声等事件

### Phase 4: 集成测试 (1天)
1. 集成到游戏主循环
2. 测试不同阶段的氛围效果
3. 调整参数平衡

## 音效资源需求

1. **使用Unity内置音效**：如 `UnityAmbient` 等
2. **程序生成简单音效**：使用AudioSource生成简单波形

## 与现有系统的集成

### 1. AudioManager集成
- 使用现有的 `PlayLoopingAudioById` 播放持续音效
- 使用 `PlaySpatialAudioById` 播放3D空间音效
- 使用 `PlaySFXById` 播放一次性音效

### 2. 事件系统集成
- 使用现有的 `EventCenter` 广播恐怖事件
- 监听游戏阶段变化事件

### 3. UI系统集成
- 在 `GameSencePanel` 中添加恐怖UI效果
- 屏幕闪烁、雪花等效果

## 验证方法

1. **音效测试**：在不同游戏阶段播放环境音，验证音量和效果
2. **视觉测试**：调整理智值，观察屏幕效果变化
3. **事件测试**：触发各种恐怖事件，验证触发逻辑和效果
4. **性能测试**：确保所有效果在目标配置下保持60FPS
